========================================
StudyHub - Two-Version Implementation Plan
========================================

PRODUCT IDEA
------------
Product: StudyHub - Collaborative Study Resource Manager
End-user: University students
Problem: Students struggle to keep track of scattered study resources (PDFs, notes, links) across multiple platforms
Solution: A centralized web application to upload, organize, and share study materials within course-specific groups

Core Feature: Upload, categorize, and share study resources with tagging and search functionality


========================================
VERSION 1 - Does One Core Thing Well
========================================

Core Value: Enable students to upload, organize, and access study materials in one centralized platform

COMPONENTS:

1. Backend (Node.js + Express)
   - User registration and login with JWT authentication
   - Create and join courses
   - Upload resources (files and links)
   - Browse and search resources
   - Download resources

2. Database (PostgreSQL)
   - Users table (credentials and profile)
   - Courses table (course information)
   - Course members table (track enrollment)
   - Resources table (uploaded files with tags)

3. Frontend (React + Bootstrap)
   - Login/Register pages
   - Dashboard (view and create courses)
   - Course page (browse and upload resources)
   - Resource detail page (view and download)

DELIVERABLE:
A functioning product where a user can:
  - Register and login
  - Create or join a course
  - Upload study materials (PDFs, documents, links)
  - Browse resources in their courses
  - Search for specific resources
  - Download resources



========================================
VERSION 2 - Builds Upon Version 1
========================================

IMPROVEMENTS TO VERSION 1:
  - Better resource preview (inline PDF viewer, image preview)
  - Enhanced search and filtering (by tags, file type, date)
  - Improved upload experience (progress indicator, better error messages)
  - Polished UI/UX (loading states, empty states, responsive design)

NEW FEATURES:
  - Comments system on resources (enables collaboration)
  - Tag-based browsing across all courses
  - User profile page (view own uploads and courses)

ADDRESS TA FEEDBACK:

DEPLOYMENT:

TECHNOLOGY STACK
========================================

Backend:    Node.js + Express.js
Database:   PostgreSQL 15
Frontend:   React + TypeScript + Bootstrap 5
Auth:       JWT (jsonwebtoken) + bcrypt
Deployment: Docker + Docker Compose
Repository: GitHub (se-toolkit-hackathon)
